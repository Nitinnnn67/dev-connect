✅ Complete Notification System Implemented
1. Created Notification Model (notification.js)

Fields: user, type, title, message, project, sender, actionUrl, isRead, createdAt
10 notification types (join_request, invitation, member_joined, etc.)
Methods: markAsRead(), getUnreadCount(), markAllAsRead()
2. Created Notification Helper (notificationHelper.js)

createNotification() - Single notification
createBulkNotifications() - Multiple notifications
Pre-built message templates
3. Updated Project Routes (project.js)
Notifications now sent when:

User requests to join project → Notifies owner
Request approved/rejected → Notifies requester
User invited → Notifies invited user
Invitation accepted/declined → Notifies inviter
New member joins → Notifies all existing members
4. Updated Notifications Route (index.js)

GET /notifications - Shows all notifications with unread count
POST /notifications/:id/read - Mark single as read
POST /notifications/mark-all-read - Mark all as read
DELETE /notifications/:id - Delete notification
5. Updated Notifications View (notifications.ejs)

Beautiful notification cards with type-specific icons
Unread indicator (colored left border)
Action buttons (View, Mark Read, Delete)
Empty state when no notifications
GSAP animations
6. Updated Navbar (navbar.ejs)

Red badge shows unread count
Only displays when count > 0
Updates on every page load via middleware
7. Added Global Middleware (app.js)

Automatically adds unread count to all views
Available as unreadNotificationCount in all EJS templates
